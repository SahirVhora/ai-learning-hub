# Content Review Process

AI resources age quickly, so every resource should carry an implicit review discipline even though the app stays static and dependency-free.

## Review Cadence

- Review high-change topics such as agents, RAG, model evaluation, and multimodal AI monthly.
- Review foundation topics quarterly.
- Remove or replace links that are unavailable, gated unexpectedly, or no longer represent the topic well.

## Acceptance Rules

- Prefer primary sources, reputable courses, papers, and hands-on tools.
- Keep each topic balanced across video, reading, and interactive resources where possible.
- Avoid adding hype-only articles unless they explain a genuinely useful concept.

## Suggested Metadata

When updating `data.js`, include review context in commit messages:

- Topic reviewed
- Resources added or removed
- Reason for any major replacement
- Date reviewed
